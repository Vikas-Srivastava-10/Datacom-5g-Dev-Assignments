Section 01

What is PDCCH?

The Physical Downlink Control Channel (PDCCH) carries Downlink Control Information (DCI) from gNB to UE. It is the most critical downlink physical channel because it tells the UE where its data is, how to decode it, and when to transmit uplink.

üì°

Primary Purpose

PDCCH carries DCI that schedules PDSCH (downlink data), PUSCH (uplink data), triggers SRS, and conveys power control commands. Without PDCCH, UE cannot know where its data is located in the frequency-time grid.

Key Innovation in NR

Unlike LTE's fixed control region (first 1‚Äì3 OFDM symbols of every subframe), NR PDCCH uses flexible CORESETs ‚Äî configurable resource blocks in both time and frequency ‚Äî enabling dynamic, beam-based control channel design.

PDCCH is always QPSK modulated (no higher-order modulation) to ensure robust reception even at cell edge. The robustness is further controlled by adjusting the Aggregation Level (AL) ‚Äî the number of CCEs used per PDCCH candidate.

Why DMRS Instead of CRS?

LTE Problem: CRS is always transmitted across full bandwidth in every subframe, regardless of whether PDCCH is present. This wastes resources and prevents UE-specific beamforming of the control channel.

NR Solution: PDCCH DMRS is only transmitted within the CORESET where PDCCH candidates exist. This enables:

‚Ä¢ Beamforming: Each CORESET can have a different beam (TCI state), so PDCCH can be beamformed to specific UEs or directions.

‚Ä¢ Energy Efficiency: No DMRS outside CORESET ‚Äî saves gNB power.

‚Ä¢ Flexible Scheduling: DMRS density is fixed per REG, but total overhead scales with CORESET size.

‚Ä¢ Multi-TRP: Different CORESETs can use DMRS from different TCI states (different transmission points).

Polar Code ‚Äî Channel Coding for PDCCH

5G NR replaces LTE's Tail-Biting Convolutional Code with Polar Codes for control channels. This provides superior performance at short block lengths.

üß¨

Polar Code Parameters

  • Mother Code LengthN = 2^n (n = 5 to 10, i.e., 32 to 1024)
  • Information Bits (K)DCI payload + 24 CRC bits
  • Coded Bits (E)AL √ó 108 (after rate matching)
  • CRC PolynomialCRC-24C: g(x) = x^24 + x^23 + x^21 + ... + 1
  • Distributed CRCCRC bits are interleaved within polar code for early termination
  • DecodingSuccessive Cancellation List (SCL) decoder with CRC-aided selection (CA-SCL)
  • List SizeTypically L=8 in commercial UEs
  • Rate MatchingRepetition (E > N), Puncturing, or Shortening (E < N)
üìà

Why Polar Code for PDCCH?

Short Block Length Performance: Polar codes approach the Shannon capacity limit at short block lengths (typical DCI is 20‚Äì140 bits), outperforming both TBCC and Turbo codes in this regime.

CRC-Aided List Decoding: The distributed CRC within the polar code enables the decoder to identify the correct codeword from a list of candidates, significantly improving BLER performance.

Low-Complexity Encoding: Polar encoding is a simple matrix multiplication with the Kronecker product of the 2√ó2 kernel matrix F = [1 0; 1 1], resulting in O(N log N) complexity.

Flexible Rate Matching: The polar code natively supports any rate through repetition, puncturing, or shortening, making it ideal for the variable DCI sizes and aggregation levels in NR.

Section 12

Beam Management for PDCCH

One of NR's most significant advances ‚Äî PDCCH can be beamformed using TCI states, enabling directed control channel transmission in FR2 (mmWave).

üì°

TCI States for PDCCH

Each CORESET can be configured with up to 64 TCI (Transmission Configuration Indicator) states. A TCI state defines the QCL (Quasi Co-Location) relationship between the PDCCH DMRS and a reference signal (SSB or CSI-RS).

QCL Types for PDCCH:

‚Ä¢ QCL-TypeA: Doppler shift, Doppler spread, average delay, delay spread

‚Ä¢ QCL-TypeD: Spatial Rx parameter (beam direction) ‚Äî most important for FR2

The active TCI state determines which beam the gNB uses to transmit PDCCH and which beam the UE uses to receive it.

üîÑ

PDCCH Beam Indication Flow

SSB-Based Initial Beam

During initial access, CORESET#0 is QCL'd with the SSB. UE uses the SSB beam for receiving PDCCH.

TCI State Configuration

Via RRC, gNB configures tci-StatesPDCCH with multiple TCI states, each pointing to a CSI-RS or SSB.

TCI State Activation

gNB activates a specific TCI state for the CORESET via MAC-CE (MAC Control Element). This changes the PDCCH beam.

Beam Failure Recovery

If PDCCH beam quality drops below threshold, UE triggers BFR procedure: detects failure via BFD-RS, selects new candidate beam, sends PRACH/dedicated SR.

‚ÑπÔ∏è
FR1 vs FR2: In FR1 (sub-6 GHz), PDCCH beamforming is optional ‚Äî many deployments use wide beams. In FR2 (mmWave, 24.25‚Äì52.6 GHz), narrow-beam PDCCH is essential due to higher path loss. Each CORESET can have a different beam, enabling multi-panel/multi-beam PDCCH monitoring.
Section 13

CCE-to-REG Mapping

How CCEs are mapped to physical REGs within the CORESET. NR supports two modes: interleaved and non-interleaved.

‚äû Interleaved Mapping

REG bundles of a CCE are distributed (scattered) across the CORESET in frequency. This provides frequency diversity.

  • DiversityHigh frequency diversity
  • BeamformingDifficult (REGs spread across bandwidth)
  • Use CaseCSS (common messages), FR1 macro cells
  • CORESET#0Always interleaved
  • Parametersreg-BundleSize, interleaverSize, shiftIndex
  • InterleaverRow-column interleaver with R rows √ó C columns

‚ñ¨ Non-Interleaved Mapping

REGs of a CCE are contiguous (localized) in frequency. This enables UE-specific beamforming.

  • DiversityLow frequency diversity (localized)
  • BeamformingExcellent ‚Äî contiguous RBs enable narrow beam
  • Use CaseUSS (UE-specific), FR2 mmWave, massive MIMO
  • REG Bundle SizeAlways 6 (entire CCE = 1 bundle)
  • PrecoderSame precoder across entire CCE ‚Äî maximum beamforming gain
  • Channel Est.Best channel estimation (all REGs contiguous)

Interleaver Operation (Row-Column Interleaving)

  Given: CORESET has N_REG_bundle REG-bundles, interleaverSize = R

  Step 1: Arrange REG-bundle indices into matrix (R rows √ó C columns):
          C = N_REG_bundle / R

           Col 0   Col 1   Col 2   ...   Col C-1
  Row 0  [  0       1       2     ...    C-1    ]
  Row 1  [  C      C+1     C+2    ...    2C-1   ]
  ...
  Row R-1[ (R-1)C  ...                   RC-1   ]

  Step 2: Read out column-by-column:
          {0, C, 2C, ..., (R-1)C, 1, C+1, 2C+1, ...}

  Step 3: Apply cyclic shift by shiftIndex (default = PCI mod N_REG_bundle)
          This randomizes mapping per cell to reduce inter-cell interference

  Result: REG bundles of each CCE are spread across different frequency positions
Section 14

Complete PDCCH Processing Chain

End-to-end signal processing from DCI generation at gNB to DCI extraction at UE.

üèóÔ∏è

gNB Transmitter Side

DCI Payload Generation

MAC scheduler generates DCI fields (frequency allocation, MCS, HARQ, TPC, etc.) based on scheduling decisions. Payload size depends on DCI format and configuration.

CRC Attachment & RNTI Masking

24-bit CRC (CRC-24C) is computed from DCI payload and appended. The last 16 bits of CRC are XORed with the target UE's RNTI.

Polar Encoding

K information bits (DCI + CRC) are encoded using polar code. Frozen bit positions determined by channel reliability sequence. Output: N = 2^n coded bits.

Rate Matching

Polar coded bits are rate-matched to produce exactly E = AL √ó 108 bits. Uses circular buffer with repetition (if E > N) or puncturing/shortening (if E < N).

Scrambling

Rate-matched bits are scrambled with pseudo-random sequence initialized by c_init = n_RNTI √ó 2^16 + n_ID, where n_ID is the PDCCH DMRS scrambling ID or PCI.

QPSK Modulation

Scrambled bits are mapped to QPSK symbols: {00 ‚Üí 1/‚àö2(1+j), 01 ‚Üí 1/‚àö2(1‚àíj), 10 ‚Üí 1/‚àö2(‚àí1+j), 11 ‚Üí 1/‚àö2(‚àí1‚àíj)}. Output: E/2 complex symbols.

CCE-to-REG Mapping

Modulated symbols are mapped to allocated CCEs within the CORESET. CCE-to-REG mapping (interleaved or non-interleaved) determines physical positions.

DMRS Insertion & Precoding

PDCCH DMRS symbols are inserted at subcarrier positions {1, 5, 9} per RB. Both DMRS and data are precoded with the same beamforming weights (from TCI state).

OFDM Signal Generation

Precoded symbols are mapped to the resource grid, IFFT is applied, CP is added, and the signal is transmitted over the air.

üì±

UE Receiver Side

CORESET Resource Identification

UE identifies the CORESET time-frequency resources based on RRC configuration (or MIB for CORESET#0).

OFDM Demodulation

Remove CP, perform FFT, extract symbols from CORESET REGs.

Channel Estimation (via DMRS)

Estimate channel from DMRS positions (k=1,5,9). Interpolate across REG bundle for data subcarriers. Apply MMSE or LS channel estimation.

Equalization & Soft Demapping

Apply channel inverse (ZF or MMSE equalization). Compute soft bits (LLRs) from received QPSK symbols.

Descrambling

Descramble soft bits using the same pseudo-random sequence as the transmitter.

Rate De-matching

Combine repeated bits, insert placeholder LLRs for punctured/shortened positions.

Polar Decoding (CA-SCL)

Apply Successive Cancellation List (SCL) decoder with list size L (typically 8). Use distributed CRC for path selection.

CRC Check & RNTI Verification

Compute CRC of decoded DCI payload. XOR with UE's RNTI(s). If remainder = 0, the DCI is successfully decoded and addressed to this UE.

DCI Interpretation

Parse the DCI fields according to the detected format. Extract scheduling information (RB allocation, MCS, HARQ, timing) and proceed with PDSCH reception or PUSCH preparation.

Optimization Strategies

CORESET Size Optimization

Balance between PDCCH capacity (more RBs/symbols = more CCEs) and PDSCH throughput (fewer CORESET symbols = more for data). Monitor CCE blocking to determine if CORESET is undersized.

AL Distribution Tuning

Optimize AL selection thresholds. Aggressive (lower AL) saves CCEs but risks PDCCH BLER. Conservative (higher AL) ensures reliability but limits capacity. Target: minimize average AL while keeping BLER < 1%.

PDCCH Power Boosting

Increase CORESET EPRE relative to PDSCH by 3‚Äì6 dB to improve PDCCH SINR. This allows using lower AL, freeing CCEs for more UEs.

Search Space Periodicity Tuning

Reduce monitoring periodicity for low-traffic UEs (saves UE power). Increase for latency-sensitive UEs. Balance between capacity and responsiveness.

Multi-CORESET Strategy

Use separate CORESETs for CSS vs USS. Dedicate a small CORESET for common messages and a larger one for UE-specific scheduling. In FR2, use different beams per CORESET for coverage.
