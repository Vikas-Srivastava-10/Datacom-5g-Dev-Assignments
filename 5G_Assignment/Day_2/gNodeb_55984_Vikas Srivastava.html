Understanding gNodeB Disaggregation

In 5G NR (New Radio), the base station, known as gNodeB, is broken down into three functional entities to allow for better flexibility, scalability, and virtualization.

1. Central Unit (CU)

The CU is a logical node that includes the gNodeB functions such as transfer of user data, mobility control, and session management. It is further split into CU-CP (Control) and CU-UP (User) for better resource management.

2. Distributed Unit (DU)

The DU is located closer to the RU. It handles the "time-critical" functions like MAC (Medium Access Control), RLC (Radio Link Control), and high-level Physical layer tasks. One CU can control multiple DUs.

3. Radio Unit (RU)

The RU is the physical hardware containing the antennas. It handles the Low-PHY layer functions, digital-to-analog conversion, and beamforming to transmit signals to the User Equipment (UE).

4. Interfaces

  • F1 Interface: Connects CU and DU.
  • Uu Interface: The radio link between RU and UE.
  • NG Interface: Connects gNodeB to the 5G Core.
