Architecture Overview: UDM in 5G

The Unified Data Management (UDM) is a critical Network Function (NF) in the 5G Core (5GC) Service-Based Architecture. It is essentially the successor to the HSS (Home Subscriber Server) from 4G LTE but is designed to be cloud-native and stateless.

Key Functions of UDM:

  • Subscription Management: Storing and managing the subscriber's profile, including their identifiers (SUPI/GPSI) and service entitlements.
  • Authentication Processing: Generating 5G authentication vectors and credentials in coordination with the AUSF.
  • Identity De-concealment: Using the Subscription Identifier De-concealing Function (SIDF) to resolve the privacy-protected SUCI back into a SUPI.
  • Mobility Management: Keeping track of the serving AMF for a user to ensure reachability.

Stateless Design & UDR:

Unlike traditional HSS, the UDM is designed to be stateless. It does not store the data itself; instead, it acts as a frontend logic layer that communicates with the UDR (Unified Data Repository) via the N35 interface. This allows for massive scalability and high availability.

Interfaces and Connectivity:

  • Nudm (Service Based): The primary interface through which other NFs (AMF, SMF, AUSF) consume UDM services.
  • N8 (UDM - AMF): Used for registration and mobility management.
  • N10 (UDM - SMF): Used for session management and subscription data retrieval.
  • N13 (UDM - AUSF): Used for authentication procedures.
